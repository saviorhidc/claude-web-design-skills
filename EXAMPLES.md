## Example 1: Vague Prompt → SPADE Prompt

### Before (generic output)

```
Build a dashboard for my analytics app.
```

**Result:** Generic white/grey layout, Inter font, blue accent buttons, no animation, no visual identity.

### After (SPADE-structured, skills-informed)

```
Build an analytics dashboard.

SPEC:
- Internal analytics dashboard for a SaaS product
- Sections: sticky sidebar nav, header with date filter, 4-up metric grid, chart section (2/3 line + 1/3 donut)
- Stack: Next.js 14 App Router, Tailwind, shadcn/ui, Framer Motion, Recharts
- Theme: dark (bg-[#0a0a0b]), surface cards bg-[#111114], emerald accent, DM Mono for numbers, Geist for UI text

PERSONA:
- Operations lead checking metrics each morning before standup
- Data-forward — wants density and signal, not whitespace
- Frustrated by Notion-style dashboards that bury numbers behind clicks

ANCHORS:
- Data density like a Bloomberg Terminal panel
- Color restraint like Grafana
- Motion subtlety like linear.app (informational, not decorative)

DONE-CRITERIA:
- All metric values readable without hover (no "click to expand")
- Renders at 320/768/1280px without horizontal scroll
- Skeletons during data load — never blank states
- Lighthouse perf >= 90 with mock data
- Tab order: filter → first metric → chart → sidebar nav (last)
- No pure white — text-white/90 primary, text-white/60 secondary

EXAMPLES:
- Match visual style of src/components/ui/Card.tsx
- Reference layout: vercel.com/analytics dashboard
- Use animation variants from src/styles/animations.ts (fadeInUp, staggerContainer)
- Recharts colors from theme tokens (no rainbow defaults)
- shadcn components: Tabs (date filter), Badge (delta — destructive if negative), Skeleton (loading)
```

**Result:** Distinctive dark dashboard with data-forward aesthetic, verifiable accessibility, consistent motion, no generic defaults.

## Example 2: Hardcoded Secret → Secure Proxy Pattern

### Before (insecure — generated by AI tools)

```typescript
// app/checkout/page.tsx
'use client';

const stripe = new Stripe('sk_live_abc123xyz...', { apiVersion: '2024-06-20' });

export default function CheckoutPage() {
  const handleCheckout = async () => {
    const session = await stripe.checkout.sessions.create({...});
    // sk_live_ key is now visible in browser devtools and git history
  };
}
```

### After (security skill applied)

```typescript
// .env.local (gitignored)
STRIPE_SECRET_KEY=sk_live_abc123xyz...
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_live_...

// app/api/checkout/route.ts — server-side only
import Stripe from 'stripe';
const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, { apiVersion: '2024-06-20' });

export async function POST(request: Request) {
  const session = await request.json();
  // auth check first
  const user = await auth();
  if (!user) return Response.json({ error: 'Unauthorized' }, { status: 401 });

  const checkoutSession = await stripe.checkout.sessions.create({
    payment_method_types: ['card'],
    line_items: session.items,
    success_url: `${process.env.NEXT_PUBLIC_APP_URL}/success`,
    cancel_url: `${process.env.NEXT_PUBLIC_APP_URL}/cancel`,
  });

  return Response.json({ url: checkoutSession.url });
}

// app/checkout/page.tsx — client-side calls the proxy
'use client';
export default function CheckoutPage() {
  const handleCheckout = async () => {
    const { url } = await fetch('/api/checkout', {
      method: 'POST',
      body: JSON.stringify({ items }),
    }).then(r => r.json());
    window.location.href = url;
  };
}
```

## Example 3: Trust-Client User ID → Server Session Auth

### Before (insecure — common AI-generated pattern)

```typescript
// app/api/profile/route.ts
export async function GET(request: Request) {
  // Trusts whatever the client sends
  const userId = request.headers.get('x-user-id');
  const profile = await db.query(`SELECT * FROM users WHERE id = '${userId}'`);
  // Two problems: client-controlled userId, string-concatenated SQL
  return Response.json(profile);
}
```

### After (security skill applied)

```typescript
// app/api/profile/route.ts
import { auth } from '@/lib/auth';
import { z } from 'zod';

export async function GET(request: Request) {
  // Identity comes from server session — client cannot influence it
  const session = await auth();
  if (!session?.user?.id) {
    return Response.json({ error: 'Unauthorized' }, { status: 401 });
  }

  // Parameterized query — no SQL injection possible
  const profile = await db.query(
    'SELECT id, name, email, created_at FROM users WHERE id = $1',
    [session.user.id]
  );

  if (!profile.rows[0]) {
    return Response.json({ error: 'Not found' }, { status: 404 });
  }

  return Response.json(profile.rows[0]);
}
```

## Example 4: Unsanitized SQL → Parameterized Query

### Before (SQL injection — AI-generated pattern)

```typescript
// Searching users by name
export async function GET(request: Request) {
  const { searchParams } = new URL(request.url);
  const query = searchParams.get('q');

  // 🚨 SQL INJECTION: attacker sends q='; DROP TABLE users; --
  const results = await db.query(
    `SELECT * FROM users WHERE name LIKE '%${query}%'`
  );
  return Response.json(results.rows);
}
```

### After (security skill applied)

```typescript
import { z } from 'zod';

const searchSchema = z.object({
  q: z.string().min(1).max(100).regex(/^[a-zA-Z0-9\s'-]+$/),
});

export async function GET(request: Request) {
  const session = await auth();
  if (!session?.user?.id) {
    return Response.json({ error: 'Unauthorized' }, { status: 401 });
  }

  const { searchParams } = new URL(request.url);
  const parsed = searchSchema.safeParse({ q: searchParams.get('q') });

  if (!parsed.success) {
    return Response.json({ error: 'Invalid query' }, { status: 400 });
  }

  // Parameterized — user input never touches the SQL string
  const results = await db.query(
    'SELECT id, name, email FROM users WHERE name ILIKE $1 LIMIT 20',
    [`%${parsed.data.q}%`]
  );

  return Response.json(results.rows);
}
```
